## 분리 3요건 (1109 문단 4.3.1 / 4.3.3) — 모두 충족 시 주계약과 분리
1. 내재파생의 경제적 특성·위험이 주계약과 **밀접히 관련되지 않음**
2. 동일조건 별도상품이 **파생상품 정의 충족**
3. 복합상품 전체를 **FVPL로 측정하지 않음** (FVPL이면 분리 안 함)

## 상품별 판정 (7상품 매핑)

### 전환권 / 신주인수권
- **자본(지분상품) 분류** 시 = compound instrument → **split accounting**
  (전체 공정가치 − 부채 = 잔여 자본). 분리요건 판단 불요.
- **파생상품부채 분류** 시 = hybrid instrument → 전환권을 주계약과 **분리**
  (주가위험이 채무 주계약과 밀접하지 않음).
- **fixed-for-fixed**(확정수량 자기지분 ↔ 확정금액) 충족→자본, 리픽싱 등 변동→파생부채.

### 상환권 (put/call/중도상환) — ★RCPS 소급 설명
- **비분리**: 매 행사시점 행사금액 ≈ 그 시점 채권 **상각후원가**(장부가). par발행·par상환 등.
- **분리**: 액면발행인데 **보장수익률로 초과수익 가산** / 행사금액이 특정금액(액면·보장원리금)이라
  상각후원가와 **중요한 차이**. (만기 근접 시점 제외)
- **★ RCPS 모비어스(8% 보장수익 풋) = 분리 대상.** host/embedded 분할이 까다로웠던 회계적 근거.
- **★★ 이중계상(double accounting) 금지**:
  - 분리 인식 → 주계약 유효이자율은 **계약만기 기준**(상환특성 유효이자율 반영 X).
  - 비분리 → **기대만기 기준**(상환특성 유효이자율 반영 O).
  - 둘 다 반영 금지. 상환권 정교화 시 필수 준수.

### 분리형 신주인수권부사채(BW) — 2.1.4.3
- 분리형 신주인수권 = 사채와 독립 양도 가능 → **내재파생 아님, 별도 금융상품**(파생상품).
- 발행 = 사채 + 신주인수권 두 별개 상품 동시발행. 각각 별도 회계·평가.
- **엔진 함의**: 분리형 = bond 독립평가 + warrant 독립평가 → **단순 합산**(상호작용 없음).
- 비분리형 = 복합상품 → 내재파생 분리요건 판단 → 격자에서 동시 고려(hold vs 행사).

### 복수 내재파생 통합 (문단 B4.3.4)
- 전환권+상환권이 상호의존적 → **"하나의 복합내재파생상품"으로 전체로서(as a whole) 평가**.
- 이것이 **혼합할인율 이항모형** = TF & Hull(지분-채권 현금가중) / GS(전환확률가중).
- **★ 우리 TF 엔진이 이 회계기준 요구를 구현 중 = 접근 정합성 확인.**
- 접근법1(밀접관련성 판단+회계처리 모두 통합) vs 접근법2(개별판단 후 분리대상만 통합) = 회계정책 선택.

### 재검토 금지 (B4.3.11)
- 분리여부는 **최초 계약시점에 판단**. 계약조건 유의적 변경 아니면 **후속 재검토 금지**.

## 엔진 반영 (BW)
- terms/rights에 `separable` 플래그로 분기:
  - `separable=true` → bond_value(독립) + warrant_value(독립 콜옵션) + dilution_effect(신주발행 희석) + redemption. **단순 합산**.
  - `separable=false` → **복합 격자**(warrant를 hold와 동시비교, 채권과 얽힘).
- **희석**: 자사 신주발행 → 기존주주 희석 → warrant_value에 희석 반영(dilution_effect 음수).
- **★ v0 범위**: separable 분기 + 희석 반영 구현, self-consistency 검증. 회계분류(자본/부채) 자체
  판정은 **사용자 입력 전제**(계약이 이미 분류).

### 페이오프 희석반영법 (3.4.1.4) — BW 핵심
- 신주인수권은 **행사시점에 투자금 납입**(발행시점 납입인 CB와 다름) → **페이오프 계산 시 희석 반영** 권장
  (사전희석반영법 아님).
- 행사 페이오프 = (기초자산가치 − strike) 계산 시 신주발행 희석계수 반영.
  희석계수 ≈ 기존주식수 / (기존주식수 + new_shares).
- dilution_effect = 희석으로 인한 가치 감소분(음수)을 **별도 컴포넌트로 분리 계상**.
