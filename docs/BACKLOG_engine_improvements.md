## 후보 목록

### 1. RCPS host/embedded 분할 재검토 ← 최우선 후보
- **현 상태**: total은 외부(모비어스) 1~2% 재현(BLOCKING). host/embedded 분할만 보고서 독점규약
  (내재 유효할인율 r_eff 27~34%)이라 non-blocking으로 강등.
- **참고 절**: 3.3.3.2(우선주 기반 복합금융상품의 조기상환청구권 평가 고려사항),
  3.3.3.3(주식발행초과금 감액 가정), 3.3.3.4(추정손익정보 고려).
- **기대**: 못 닫은 host/embedded 분할(측도분리 C모델 한계)이 이 절의 우선주기반 상환권
  평가법으로 설명·해소될 가능성. r_eff 27~34%의 정체가 여기 있을 수 있음.
- **조치(나중)**: 3.3.3.2 정독 → C모델과 대조 → 분할이 닫히면 RCPS 개선, 아니면
  "우리 방식과 보고서 방식의 회계정책 차이"로 limitations 근거 보강.
- **회계적 단서(분리판정 규칙 문서 참조)**: RCPS 모비어스 풋(8% 보장수익)은 **분리 대상**
  (액면발행+보장수익률 초과가산이라 상각후원가와 중요한 차이). host/embedded 분할이
  까다로웠던 회계적 근거가 여기 있음. → `docs/KIFRS_분리판정_규칙.md` 상환권 절 참조.

### 2. 상환권 분리/비분리 & 이중계상 방지
- **참고 절**: 2.1.3.2(상환권 분리요건), 3.3.2/3.3.3(조기상환청구권 평가).
- **핵심 규칙**(→ `docs/KIFRS_분리판정_규칙.md`): 분리 인식→주계약 유효이자율 **계약만기 기준** /
  비분리→**기대만기 기준**. **이중계상(double accounting) 금지**.
- **조치(나중)**: CB·RCPS·EB의 상환권 처리가 이 규칙과 일관되는지 점검. 현재는 격자에 상환
  직접 반영(분리 방식) — 유효이자율 이중반영 없는지 확인.

### 3. CB 희석 방법 정합성
- **참고 절**: 3.4.1(희석효과 반영 방법: 통합적/내재적/페이오프).
- **현 상태**: CB dilution 컴포넌트 있음(자사주 전환). BW에서 페이오프 희석반영법 도입 후,
  CB 희석도 같은 프레임으로 일관되는지 점검 후보.
- **조치(나중)**: BW 완료 후 CB 희석과 방법론 통일 검토.

### 4. refixing 정교화
- **참고 절**: 3.6(리픽싱 + 경로의존성 + FSGM/MC/수정이항).
- **현 상태**: 이벤트형 refixing을 "미발동"으로 단순 처리(모비어스·DeepSearch 모두 미발동이었음).
- **조치(나중)**: 리픽싱 발동 케이스 지원 시 3.6 기반 경로의존 평가 도입.

### 5. GS 모델 옵션 (별도 백로그와 연계)
- **참고 절**: 3.2.2.3 GS 모형(전환확률가중). `golden-values/rcps_gs_deepsearch_case.json`(참고값, **현재 미생성** — 정리단계 대상).
- **조치(나중)**: TF/GS 선택 옵션 구현 시 3.2.2.3 근거.

## 원칙
- 소급 수정은 **반드시 정리단계에서**, 기존 검증값(RCPS total 등) 불변 확인하며.
- 각 개선은 **독립 슬라이스로**(한 번에 하나). 개선 전후 **CI green 유지**.
